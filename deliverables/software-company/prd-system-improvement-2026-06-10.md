## 1. 原始需求复述

OCRScanStruct 是医疗损害责任纠纷案件的证据整理与起诉状生成平台。当前系统有 4 步工作流（创建案件 → 上传素材 → 证据目录 → 分析与导出）。本次需求包含 4 大项改造：

1. **智能分析按钮改造** — 分析完成后按钮 disabled，新增"再次智能分析"按钮支持覆盖重跑
2. **步骤导航重构** — 案件中点击步骤不跳出，侧边栏入口与创建案件分离
3. **案件类型合并** — 取消新生儿独立类型，合并为伤残/死亡 + 未成年人开关，模板从 6 套缩减为 4 套
4. **新增"赔偿金额计算"步骤** — 在上传素材和证据目录之间插入新步骤，自动提取医疗费用并按法律规定计算各项赔偿

---

## 2. 产品目标

1. **降低误操作风险**：通过智能分析按钮状态管理和步骤导航保护，防止用户在案件中意外跳出到新建界面丢失状态
2. **简化案件类型体系**：将 3 类 × 2 开关 = 6 模板精简为 2 类 × 2 开关 = 4 模板，减少用户选择负担和维护成本
3. **增强赔偿计算能力**：提供结构化的赔偿金额计算功能，自动从素材提取费用数据并按法定公式计算，提升起诉状质量与效率

---

## 3. 用户故事

### 需求1：智能分析按钮改造

- **US-1.1**: 作为律师用户，我希望点击"开始智能分析"后按钮自动变灰不可再次点击，以免重复触发分析
- **US-1.2**: 作为律师用户，我希望分析完成后出现"再次智能分析"按钮，这样在系统代码更新后可以用新逻辑重新分析已有素材
- **US-1.3**: 作为律师用户，我希望点击"再次智能分析"时系统明确提示会覆盖原有分析结果，以免误操作丢失数据

### 需求2：步骤导航重构

- **US-2.1**: 作为律师用户，我希望在案件工作流中点击步骤条时留在当前案件内切换步骤，而不是跳出到新建案件界面
- **US-2.2**: 作为律师用户，我希望侧边栏"诉状生成"入口默认展示已有案件列表，而不是空白创建界面
- **US-2.3**: 作为律师用户，我希望有一个独立的"创建案件"按钮来开始新案件，与继续已有案件的操作路径清晰分离

### 需求3：案件类型合并

- **US-3.1**: 作为律师用户，我希望案件类型只有"伤残"和"死亡"两个选项，通过"是否未成年人（新生儿）"开关来区分成年/未成年/新生儿案件，简化创建流程
- **US-3.2**: 作为律师用户，当选择伤残并打开未成年人开关时，系统应自动加载包含法定代理人信息的未成年人伤残模板
- **US-3.3**: 作为律师用户，当选择死亡并打开未成年人开关时，系统应自动加载包含法定代理人信息的未成年人死亡模板

### 需求4：新增"赔偿金额计算"步骤

- **US-4.1**: 作为律师用户，我希望上传素材后系统能自动从医疗费用类素材中提取和计算各项赔偿费用，减少手工计算工作量
- **US-4.2**: 作为律师用户，我希望能够查看和对应原始素材（发票、清单等），以核实每项费用的来源
- **US-4.3**: 作为律师用户，我希望可以手动调整和补充系统自动计算的赔偿费用，以处理特殊情况
- **US-4.4**: 作为律师用户，我希望可以将赔偿计算结果导出为 Excel，方便在起诉状中引用

## 6. 关键技术影响分析

### 6.1 步骤号变更影响

当前步骤编号与新增步骤后的映射：

| 当前 | 新增后 | 步骤名称 |
|------|--------|----------|
| Step 0 | Step 0 | 创建案件 |
| Step 1 | Step 1 | 上传素材 |
| - | **Step 2** | **赔偿金额计算（新增）** |
| Step 2 | Step 3 | 证据目录 |
| Step 3 | Step 4 | 分析与导出 |

**影响范围**：
- 前端 `currentStep` 所有硬编码的步骤号引用需更新
- `navigateToStep()`、`goStep2()`、`goStep3()` 等函数
- 进度轮询完成后的自动跳转逻辑（`startProgressPoll` 中跳到 step 2/3 的逻辑）
- `continueCase()` 中根据案件状态决定进入哪个步骤
- `handleHashChange()` 中的步骤范围校验（当前 `step >= 0 && step <= 3` 需改为 `<= 4`）
- URL hash `step=N` 格式需同步更新

### 6.2 模板系统变更

**当前**：6 套模板 → `get_template_key()` 支持 3 种 case_type × 2 种 is_minor
**目标**：4 套模板 → 移除 neonatal_adult 和 neonatal_minor

`get_template_key()` 修改：
```python
# 修改前
if case_type == "neonatal":
    return "neonatal_minor" if is_minor else "neonatal_adult"

# 修改后：删除 neonatal 分支
# 旧 neonatal 案件映射为 injury + is_minor=True
```

### 6.3 数据库兼容

- `case_type` 字段现有值 `neonatal` 的记录需迁移为 `injury`，同时 `is_minor` 设为 `True`
- Schema 中 `Literal["injury","death","neonatal"]` 改为 `Literal["injury","death"]`
- 前端 `caseTypeLabel()` 函数移除 neonatal 分支
